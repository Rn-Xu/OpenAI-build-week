# Judge Q&A preparation

## What makes this different from a to-do list?

Most to-do lists organize tasks the user has already defined. Get up! focuses on the moment before action. It translates an overwhelming goal into one immediate step and adjusts that step when the user cannot begin.

## Why does this need AI?

The same task can feel difficult for different reasons. Natural-language understanding lets Get up! respond to the user's goal and feedback instead of forcing every user through a fixed template. The AI is constrained to generate a concrete, short, testable next action.

## How do you prevent generic advice?

The response format requires an observable action, a short time estimate, and a completion condition. Suggestions such as “work harder” or “stay focused” do not satisfy the format.

## How do you measure whether it works?

The first meaningful metric is task-start rate: whether a user completes the first suggested action. Future testing can compare normal plans with progressively smaller steps and measure completion, time-to-start, and self-reported overwhelm.

## What about mental-health safety?

Get up! is a productivity aid, not a medical product or substitute for professional care. It should avoid diagnosis, shame, coercion, or high-pressure language, and clearly direct users toward appropriate support when a request indicates risk of harm.

## What did Codex contribute?

Codex helped turn the initial concept into a structured prototype: shaping the interface, refining prompts and outputs, debugging implementation details, and preparing the submission and demo workflow.

## What would you build next?

First, we would test the core interaction with real users. Only after validating that it improves task-start rates would we add personalization, voice, calendar integration, reminders, and progress history.
