# Final submission checklist

## Accuracy

- [ ] Every feature described in **What it does** is visible in the working Demo.
- [ ] Every technology tag was actually used.
- [ ] Future features appear only in **What's next**.
- [ ] Mockups have been replaced with real screenshots where possible.
- [ ] The story does not claim user results that were never measured.

## Links

- [ ] Live Demo opens in a private/incognito browser without special access.
- [ ] GitHub repository is public if included.
- [ ] Repository history and files contain no API keys, tokens, `.env` files, personal data, or test credentials.
- [ ] README contains setup and run instructions.
- [ ] YouTube/Vimeo/Youku video opens without signing in.
- [ ] Video is Unlisted/Public, not Private.

## Media

- [ ] Cover is the first image.
- [ ] All images use a 3:2 ratio and are under 5 MB.
- [ ] Screenshots contain no notifications, emails, private tabs, local paths, or credentials.
- [ ] Video is approximately two minutes and shows real product operation.
- [ ] English captions have been reviewed for errors.
- [ ] Audio is clear and background music does not overpower narration.

## Copy

- [ ] Replace every `[ADD ...]` placeholder.
- [ ] Confirm the project name is consistently written as **Get up!**.
- [ ] Preview Markdown headings, lists, and quotation blocks.
- [ ] Read the final story aloud once for clarity.
- [ ] Save the Devpost draft before the deadline.
